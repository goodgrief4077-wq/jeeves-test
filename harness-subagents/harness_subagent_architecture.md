# Collaborative Subagent Framework Architecture
## Harness Engineering Implementation for Top 1% Quality

### Overview
Multi-agent collaborative framework designed to achieve elite quality standards through specialized expertise and systematic collaboration.

### Core Principles
- **Conservative Scoring**: Objective benchmarking against top industry standards
- **Specialized Expertise**: Dedicated agents for quality domains
- **Collaborative Workflows**: Systematic data sharing and verification
- **Harness Engineering**: Constraint systems and verification loops

### Subagent Roles

#### 1. Harness Architect (Primary)
- **Role**: System design and coordination
- **Responsibilities**: Architecture definition, workflow orchestration, integration management
- **Expertise**: System design, constraint engineering, collaborative workflows

#### 2. Quality Auditor
- **Role**: Objective benchmarking and quality assessment
- **Responsibilities**: Performance scoring, security assessment, accessibility verification, code quality evaluation
- **Expertise**: Industry benchmarking, objective metrics, quality standards

#### 3. Performance Specialist
- **Role**: Runtime optimization and performance engineering
- **Responsibilities**: Load testing, latency optimization, resource efficiency
- **Expertise**: Performance profiling, bottleneck analysis, optimization strategies

#### 4. Security Hardener
- **Role**: Security implementation and threat mitigation
- **Responsibilities**: Vulnerability assessment, security controls, threat modeling
- **Expertise**: Security frameworks, penetration testing, secure coding

#### 5. Accessibility Expert
- **Role**: Universal access and inclusivity
- **Responsibilities**: WCAG compliance, user experience testing, assistive technology support
- **Expertise**: Accessibility guidelines, usability testing, inclusive design

### Collaboration Protocols

#### Data Sharing Framework
- **Standardized Formats**: JSON schemas for quality assessments
- **Verification Loops**: Cross-agent validation of findings
- **Aggregate Scoring**: Weighted quality scores across domains

#### Quality Assessment Methodology
```json
{
  "scoring_categories": ["performance", "security", "accessibility", "code_quality"],
  "benchmark_levels": {
    "top_1%": ">95",
    "top_10%": "90-95", 
    "top_25%": "80-89",
    "average": "70-79",
    "needs_improvement": "<70"
  }
}
```

#### Interaction Patterns
1. **Sequential Assessment**: Quality Auditor → Performance Specialist → Security Hardener → Accessibility Expert
2. **Parallel Evaluation**: Specialized agents assess independently with subsequent integration
3. **Consensus Validation**: Collaborative review and score adjustment

### Technical Implementation
- **Prompt Profiles**: Specialized system prompts for each agent type
- **Data Persistence**: Shared storage for assessment results
- **API Contracts**: Standardized interfaces between agents
- **Quality Dashboard**: Consolidated reporting and visualization

### Quality Framework Integration
- **Objective Comparison**: Industry-standard benchmarks (Lighthouse, OWASP, WCAG)
- **Conservative Scoring**: Penalties for overstatement, validation requirements
- **Disclosure Requirements**: Mandatory disclaimers and limitations
